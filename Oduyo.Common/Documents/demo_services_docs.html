🎯 IDemoService

💡 Demo Nedir? Potansiyel müşterilere yapılacak ürün tanıtım toplantılarının planlandığı ve takip edildiği sistemdir.

Demo Planlama - CreateDemoAsync

Web API Controller

[HttpPost]
public async Task<IActionResult> CreateDemo([FromBody] CreateDemoDto dto)
{
    var currentUserId = int.Parse(User.FindFirst(ClaimTypes.NameIdentifier).Value);
    
    var demo = await _demoService.CreateDemoAsync(dto, currentUserId);
    return Ok(new { message = "Demo planlandı", data = demo });
}

DemoStatus Enum Değerleri

Değer Açıklama
Planned (1) Planlandı - İlk durum
Completed (2) Tamamlandı - Demo yapıldı
Cancelled (3) İptal Edildi - Demo yapılmadı
Postponed (4) Ertelendi - Yeni tarihe alındı

💬 IDemoFeedbackService

💡 Demo Feedback Nedir? Demo sonrası müşteriden alınan geri bildirimlerdir. Her demo için bir feedback kaydı tutulur.

Feedback Oluşturma - CreateFeedbackAsync

var dto = new CreateDemoFeedbackDto
{
    DemoId = 5,
    Feedback = "Müşteri ürünleri çok beğendi. Online Tahsilat paketine ilgi gösterdi. Fiyat teklifi talep etti."
};

try
{
    var feedback = await _demoFeedbackService.CreateFeedbackAsync(dto, currentUserId);
}
catch (InvalidOperationException ex)
{
    // Bu demo için zaten feedback mevcut
}
⚠️ Önemli: Her demo için sadece bir feedback kaydı tutulur. Mevcut feedback'i güncellemek için UpdateFeedbackAsync kullanın.

Feedback Güncelleme - UpdateFeedbackAsync

var dto = new UpdateDemoFeedbackDto

📋 Servis Özet Tablosu

Servis Metod Açıklama Dönüş Tipi
IDemoService CreateDemoAsync Yeni demo planlar Task<Demo>
UpdateDemoAsync Demo bilgilerini günceller Task<Demo>
DeleteDemoAsync Demo'yu siler (soft delete) Task<bool>
GetDemosByCompanyAsync Firmaya göre demoları getirir Task<List<Demo>>
GetDemosByStatusAsync Statüye göre demoları getirir Task<List<Demo>>
GetDemosByAssignedUserAsync Atanan kullanıcıya göre getirir Task<List<Demo>>
GetUpcomingDemosAsync Yaklaşan demoları getirir Task<List<Demo>>
CompleteDemoAsync Demo'yu tamamlar Task<bool>
CancelDemoAsync Demo'yu iptal eder Task<bool>
PostponeDemoAsync Demo'yu erteler Task<bool>
IDemoFeedbackService CreateFeedbackAsync Yeni feedback oluşturur Task<DemoFeedback>
UpdateFeedbackAsync Feedback günceller Task<DemoFeedback>
DeleteFeedbackAsync Feedback siler Task<bool>
GetDemoFeedbackAsync Demo feedback'i getirir Task<DemoFeedback>

5. Haftalık Demo Raporu

// Bu hafta tamamlanan demoların raporu
var weekStart = DateTime.UtcNow.Date.AddDays(-(int)DateTime.UtcNow.DayOfWeek);
var weekEnd = weekStart.AddDays(7);

var completedDemos = await _demoService.GetDemosByStatusAsync(DemoStatus.Completed);

var weeklyDemos = completedDemos
    .Where(d => d.DemoDate >= weekStart && d.DemoDate < weekEnd)
    .ToList();

Console.WriteLine($"Bu Hafta Tamamlanan Demo Sayısı: {weeklyDemos.Count}\n");

var withFeedback = 0;
var withOffer = 0;

foreach (var demo in weeklyDemos)
{
    var feedback = await _demoFeedbackService.GetDemoFeedbackAsync(demo.Id);
    if (feedback != null) withFeedback++;
    
    var offers = await _offerService.GetOffersByCompanyAsync(demo.CompanyId);
    if (offers.Any()) withOffer++;
}

Console.WriteLine($"Feedback Girilmiş: {withFeedback}");
Console.WriteLine($"Teklif Oluşturulmuş: {withOffer}");
Console.WriteLine($"Dönüşüm Oranı: %{(withOffer * 100.0 / weeklyDemos.Count):F1}");

🔒 Güvenlik ve En İyi Pratikler

1. Demo Durumu Kontrolü

Demo'yu tamamlamadan önce gerçekten yapıldığından emin olun. Feedback zorunlu tutulabilir.

2. Tarih Kontrolü

Geçmiş tarihe demo planlanmasını engelleyin veya uyarı verin.

if (dto.DemoDate < DateTime.UtcNow)
{
    return BadRequest("Geçmiş tarihe demo planlayamazsınız");
}

3. Bildirimler

Demo planlandığında, ertelendiğinde veya iptal edildiğinde ilgili kişilere bildirim gönderin.

4. Demo Feedback Zorunluluğu

Demo tamamlandıktan sonra feedback girişi zorunlu tutulabilir.

var feedback = await _demoFeedbackService.GetDemoFeedbackAsync(demoId);
if (feedback == null)
{
    return BadRequest("Demo tamamlanmış ancak feedback girilmemiş");
}
✓ İş Akışı Kuralları:
  • Her demo mutlaka bir firmaya bağlı olmalı
  • Demo planlandığında statü otomatik "Planned" olmalı
  • Demo tamamlandıktan sonra feedback girilmeli
  • Feedback girilmeden teklif oluşturulamamalı (opsiyonel kural)
  • İptal edilen demolar için iptal nedeni kaydedilebilir

📊 Demo İş Akışı

Tipik Demo Süreci:

  1. Demo Talebi: Müşteri web sitesi veya çağrı merkezi üzerinden demo talep eder
  2. Firma Kaydı: Firma ve iletişim kişisi sisteme kaydedilir
  3. Demo Planlanır: Satış temsilcisine atanır, tarih belirlenir (Status: Planned)
  4. Demo Yapılır: Belirlenen tarihte demo gerçekleştirilir
  5. Demo Tamamlanır: Demo tamamlanır olarak işaretlenir (Status: Completed)
  6. Feedback Girilir: Müşteriden alınan geri bildirim kaydedilir
  7. Teklif Hazırlanır: Gerekirse teklif oluşturulur
  8. Takip: Müşteri ile takip görüşmeleri yapılır
